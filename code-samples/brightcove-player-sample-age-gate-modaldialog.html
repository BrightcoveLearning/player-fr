Exemple de lecteur

Cet exemple et le code associé sont fournis comme guide pour le développement de votre production. Le code n'est pas certifié pour une utilisation en production ni officiellement pris en charge.

Cet exemple utilise ModalDialog pour couvrir initialement le lecteur. Lorsque l'utilisateur confirme son âge en cliquant sur la fenêtre modale, les champs sont affichés et la vidéo est lue. Testez la fonctionnalité en cliquant n'importe où dans la fenêtre modale.

Séquence de développement

Voici la séquence de développement recommandée :

  1. Utilisez l'implémentation du lecteur intégré In-Page pour tester les fonctionnalités de votre lecteur, de votre plugin et de votre CSS (si CSS est nécessaire)
  2. Placez le JavaScript et le CSS du plugin dans des fichiers séparés pour des tests locaux
  3. Déployez le code du plugin et le CSS sur votre serveur une fois que vous avez corrigé des erreurs.
  4. Utilisez Studio pour ajouter le plugin et le CSS à votre lecteur
  5. Remplacez l'implémentation du lecteur intégré dans la page si vous déterminez que l'implémentation iframe est mieux adaptée (détaillée dans la section suivante)

Pour plus de détails sur ces étapes, consultez la section étape par étape : Guide de développement de plug-ins.

Intégrer iFrame ou In-Page

Lorsque vous développez des améliorations pour Brightcove Player, vous devrez décider si le code convient le mieux à l'implémentation intégrée iframe ou In-Page. La recommandation des meilleures pratiques consiste à créer un plugin à utiliser avec une implémentation iframe. Les avantages de l'utilisation du lecteur iframe sont les suivants :

  • Aucune collision avec JavaScript et/ou CSS existants
  • Réactif automatiquement
  • L'iframe facilite l'utilisation dans les applications de médias sociaux (ou lorsque la vidéo devra « voyager » dans d'autres applications)

Bien que l'intégration du lecteur d'intégration dans la page puisse être plus complexe, il y a des moments où vous planifiez votre code autour de cette implémentation. Pour généraliser, cette approche est préférable lorsque la page contenante doit communiquer avec le lecteur. En particulier, voici quelques exemples :

  • Le code dans la page contenant doit écouter et agir sur les événements du joueur
  • Le lecteur utilise des styles de la page contenante
  • L'iframe entraînera l'échec de la logique de l'application, comme une redirection à partir de la page contenant

Même si votre implémentation finale n'utilise pas le code iframe, vous pouvez toujours utiliser le code intégré In-Page avec un plugin pour votre JavaScript et un fichier séparé pour votre CSS. Cela encapsule votre logique afin que vous puissiez facilement l'utiliser dans plusieurs joueurs.

Avantages modaux

Quelques avantages à utiliser le Boîte de dialogue Modale composant comprend les éléments suivants :

  • Il couvre l'ensemble du joueur par défaut.
  • Il dispose de méthodes permettant d'ouvrir et de fermer facilement la fenêtre modale

Cet exemple utilise le composant ModalDialog du lecteur Brightcove. Pour plus de détails, consultez le document Implémentation de la Dialogule Modal.

Configuration du lecteur/HTML

Aucune configuration particulière n'est requise pour le Brightcove Player que vous créez pour cet exemple.

Flux d'application

La logique de base derrière cette application est :

  • Afficher ModalDialog cliquable avec du texte.
  • Lorsque l'utilisateur clique sur le texte ModalDialog, fermez le ModalDialog et démarrez la lecture vidéo.

Afficher le texte ModalDialog

Le ModalDialog s'affiche dès que la vidéo est chargée. Le texte de la porte d'âge est défini comme une option pour le ModalDialog.

Configurer le ModalDialog

Trouvez le code qui est étiqueté :

// +++ Configure the ModalDialog +++

Créez un objet d'options et définissez les propriétés du Classe ModalDialog. Pour plus de détails sur cet objet, consultez le Implémentation du ModalDialog document. Ici, nous faisons ce qui suit:

  • Définir le texte de la barrière d'âge
  • Définir la boîte de dialogue comme temporaire afin que nous puissions la fermer
  • Masquer le bouton de fermeture dans la boîte de dialogue

Instanciez et ouvrez le ModalDialog

Trouvez le code qui est étiqueté :

// +++ Instantiate and open the ModalDialog +++

Instanciez le dialogue avec notre texte de porte d'âge, ajoutez-le au lecteur et ouvrez le dialogue.

Écoutez l'événement de clic sur le ModalDialog

Trouvez le code qui est étiqueté :

// +++ Listen for the click event on the ModalDialog +++

Au ModalDialog, ajoutez un écouteur d'événements pour le click un événement. Lorsqu'un utilisateur sélectionne le texte de la barrière d'âge, fermez la boîte de dialogue et démarrez la lecture vidéo.

Style d'application

Le CSS stylise et positionne le texte de la barrière d'âge dans le ModalDialog.

Code du plugin

Normalement, lors de la conversion du JavaScript en plugin Brightcove Player, des modifications nominales sont nécessaires. Une modification requise consiste à remplacer l'utilisation standard de la ready() méthode par le code qui définit un plugin.

Voici le code de début de JavaScript très couramment utilisé qui fonctionnera avec le lecteur :

videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  ...
});

Vous allez modifier la première ligne pour utiliser la syntaxe standard pour démarrer un plugin Brightcove Player :

videojs.registerPlugin('pluginName', function(options) {
  var myPlayer = this;
  ...
});

Comme mentionné précédemment, vous pouvez voir le code JavaScript du plugin dans le référentiel GitHub correspondant de ce document: age-gate-modal.js.

Utilisation du plugin avec un lecteur

Une fois que les fichiers CSS et JavaScript du plugin sont stockés dans un emplacement accessible à Internet, vous pouvez utiliser le plugin avec un lecteur. Dans les studios JOUEURS module, vous pouvez choisir un joueur, puis dans le PLUGINS section ajouter les URL aux fichiers CSS et JavaScript, et également ajouter le Nom et Options , si des options sont nécessaires.

Pour plus d'informations sur l'utilisation de Studio pour attribuer des plugins aux lecteurs, consultez le Configuration des plugins de lecteur document.
