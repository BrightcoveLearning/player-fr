    Exemple de joueur Brightcove : Plugin de superposition de logo

    Dans cette rubrique, vous apprendrez à créer un plugin de superposition de logo qui utilise le plugin Overlay standard pour afficher une superposition de logo pour les vidéos, avec une URL de destination facultative. Si vous prévoyez d'utiliser le plugin avec une implémentation iframe du lecteur, vous ne pouvez pas utiliser l'URL de destination car la nouvelle page ne s'ouvrira que dans l'iframe.

    Exemple de lecteur

    Remarque : Des fonctionnalités similaires à celles présentées dans cet exemple font désormais partie de Studio, comme indiqué dans le Styliser les joueurs document. Cet exemple offre plus de contrôle sur l'emplacement du logo, l'heure d'affichage et d'autres options.
    Cet exemple et le code associé sont fournis comme guide pour le développement de votre production. Le code n'est pas certifié pour la production et n'est pas officiellement pris en charge.

    Le plugin peut être utilisé pour afficher n'importe quelle superposition d'image prête pour le Web sur le dessus du lecteur. Vous pouvez contrôler l'affichage du message en fonction:

    • Emplacement - l'image peut être affichée à différents endroits sur le lecteur
    • Événements de joueur - tels que play, pause ou des événements personnalisés
    • Intervalle de temps - à une heure spécifiée pendant la lecture vidéo

    Dans l'exemple suivant, vous voyez un logo affiché en bas à gauche du lecteur. À cinq secondes de la vidéo, le logo s'estompe lentement.

    Voir le stylo Plugin de superposition de logo par Brightcove Learning Services (@bcls ) sur CodePen.

    Utilisation du CodePen

    Voici quelques conseils pour utiliser efficacement le CodePen ci-dessus :

    • Basculez l'affichage réel du joueur en cliquant sur le Result bouton.
    • Cliquez sur les HTML/CSS/JS boutons pour afficher L'UN des types de code.
    • Plus loin dans ce document, la logique, le flux et le style utilisés dans l'application seront abordés dans les sections Configuration lecteur/HTML, Flux d' application et Style de l'application. La meilleure façon de suivre les informations contenues dans ces sections est de :
      1. Cliquez sur le EDIT ON CODEPEN bouton dans le CodePen et ayez le code disponible dans un onglet navigateur/navigateur.
      2. Dans CodePen, ajustez le code que vous souhaitez afficher. Vous pouvez modifier la largeur des différentes sections de code dans CodePen.
      3. Affichez les sections Lecteur/HTML, Flux d'application et/ou Style de l'application dans un autre onglet navigateur/navigateur. Vous serez maintenant en mesure de suivre les explications du code et en même temps voir le code.

    Séquence de développement

    Voici la séquence de développement recommandée :

    1. Utilisez l'implémentation du lecteur intégré dans la page pour tester les fonctionnalités de votre lecteur, plugin et CSS (si CSS est nécessaire)
    2. Placez le JavaScript et le CSS du plugin dans des fichiers séparés pour les tests locaux
    3. Déployez le code plugin et CSS sur votre serveur une fois que vous avez éliminé les erreurs
    4. Utilisez Studio pour ajouter le plugin et CSS à votre lecteur
    5. Remplacez l'implémentation du lecteur intégré dans la page si vous déterminez que l'implémentation iframe est mieux adaptée (détaillée dans la section suivante)

    Pour plus de détails sur ces étapes, consultez la section étape par étape : Guide de développement de plug-in .

    iframe ou incorporer dans la page

    Lorsque vous développez des améliorations pour Brightcove Player, vous devrez décider si le code convient le mieux à l'implémentation iframe ou In-Page. La meilleure pratique est de construire un plugin pour une utilisation avec une implémentation iframe. Les avantages de l'utilisation du lecteur iframe sont :

    • Aucune collision avec JavaScript et/ou CSS existants
    • Réactif automatiquement
    • L'iframe facilite l'utilisation dans les applications de médias sociaux (ou chaque fois que la vidéo aura besoin de « voyager » dans d'autres applications)

    Bien que l'intégration du lecteur d'intégration dans la page puisse être plus complexe, il y a des moments où vous planifiez votre code autour de cette implémentation. Pour généraliser, cette approche est la meilleure lorsque la page contenant doit communiquer avec le joueur. Plus précisément, voici quelques exemples :

    • Le code dans la page contenant doit écouter et agir sur les événements du joueur
    • Le lecteur utilise les styles de la page contenant
    • L'iframe provoquera l'échec de la logique de l'application, comme une redirection à partir de la page contenant

    Même si votre implémentation finale n'utilise pas le code iframe, vous pouvez toujours utiliser le code intégré In-Page avec un plugin pour votre JavaScript et un fichier séparé pour votre CSS. Cela encapsule votre logique afin que vous puissiez facilement l'utiliser dans plusieurs joueurs.

    Configuration du lecteur/HTML

    Cette section détaille toute configuration spéciale nécessaire lors de la création du joueur. En outre, d'autres éléments HTML qui doivent être ajoutés à la page, au-delà du code d'implémentation du lecteur intégré dans la page, sont décrits.

    Configuration du joueur

    Aucune configuration particulière n'est requise pour le Brightcove Player que vous créez pour cet exemple.

    Autres HTML

    le Plug-in de superposition d'affichage Le fichier JavaScript est lié à la page à l'aide d'un <script> marque. Notez que le CSS pour le Plug-in de superposition d'affichage n'est PAS lié à la page. Le CSS personnalisé est utilisé dans la page pour faciliter les modifications.

    Flux d'application

    La logique de base derrière cette application est :

    • Une fonction est définie qui supprime l'image de superposition au bon moment.
    • Une fonction est définie qui affiche l'image de superposition au bon moment.
    • Les options de configuration personnalisées sont fusionnées avec les options par défaut. Les options par défaut ne sont utilisées que si une option personnalisée n'est pas fournie.
    • La fonction est appelée qui affiche l'image de superposition.

    Supprimer la superposition

    Trouvez le code qui est étiqueté :

    // ### Remove overlay ###

    Cette fonction est appelée lorsque l'image de superposition est initialement affichée. La logique de la fonction vérifie quand l'image de superposition doit être supprimée, y compris si une heure d'expiration a été définie. Également timeupdate l'écouteur d'événements est supprimé.

    Ajouter une superposition

    Trouvez le code qui est étiqueté :

    // ### Add overlay function ###

    Le plugin Overlay est appelé ici en utilisant les paramètres de configuration utilisateur. Les paramètres déterminent ce qui doit être affiché, son emplacement et sa durée. le timeupdate un écouteur d'événement est ajouté pour vérifier une heure d'expiration.

    Construire un overlayOptions objet

    Trouvez le code qui est étiqueté :

    // ### Call function to merge default options with passed in options ###

    Les options de configuration personnalisées et les options par défaut sont transmises à une fonction. La fonction utilise les options par défaut uniquement si une option personnalisée correspondante n'est pas transmise.

    Appeler le showoverlay méthode

    Trouvez le code qui est étiqueté :

    // ### Show overlay ###

    La fonction qui affiche l'image de superposition est appelée. Selon les options, l'image peut être affichée immédiatement ou ultérieurement.

    Style d'application

    Le long CSS définit à quoi doit ressembler l'image de superposition, ainsi que l'endroit où l'image est affichée. Le CSS par défaut pour le Plug-in de superposition d'affichage n'est PAS utilisé pour faciliter la personnalisation CSS.

    Code du plugin

    Normalement, lors de la conversion du JavaScript en plug-in Brightcove Player, des changements nominaux sont nécessaires. Un changement nécessaire consiste à remplacer l'utilisation standard de la ready() méthode par le code qui définit un plugin.

    Voici le code JavaScript très couramment utilisé qui fonctionnera avec le lecteur :

    videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  ...
});

    Vous allez changer la première ligne pour utiliser la syntaxe standard pour démarrer un plugin Brightcove Player :

    videojs.registerPlugin('pluginName', function(options) {
  var myPlayer = this;
  ...
});

    Comme mentionné précédemment, vous pouvez voir le code JavaScript du plugin dans le référentiel GitHub correspondant de ce document: logo-overlay.js.

    Utilisation du plugin avec un lecteur

    Une fois que vous avez les fichiers CSS et JavaScript du plugin stockés dans un emplacement accessible à Internet, vous pouvez utiliser le plugin avec un lecteur. Dans le module PLAYERS de Studio, vous pouvez choisir un lecteur, puis dans la section PLUGINS ajouter les URL aux fichiers CSS et JavaScript, et ajouter également le nom et les options , si des options sont nécessaires. Dans le cas de ce plugin, les options seront très probablement transmises au plugin, sous la forme:

    videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  myPlayer.logoOverlay(
    {
      'align': 'bottom-left',
      'imageURL' : 'https://solutions.brightcove.com/bcls/brightcove-player/logo-overlay/bc-logo.png',
      'clickThruURL': 'https://support.brightcove.com/',
      'start': 'loadstart',
      'end': 5
    }
  )
});
    Pour plus d'informations sur l'utilisation de Studio pour attribuer des plugins aux lecteurs, consultez le document Configuration des plugins du lecteur .
