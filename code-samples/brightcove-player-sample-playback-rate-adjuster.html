Exemple de lecteur

Cet exemple et le code associé sont fournis comme guide pour le développement de votre production. Le code n'est pas certifié pour une utilisation en production ni officiellement pris en charge.

Démarrez la lecture vidéo et sélectionnez le bouton de fréquence de lecture dans la barre de contrôle (la valeur par défaut est 1x). Choisissez un taux différent pour voir comment la vitesse de lecture vidéo change. Notez que lorsque vous modifiez la vitesse de lecture, la durée affichée est recalculée en fonction de la vitesse sélectionnée.

Séquence de développement

Voici la séquence de développement recommandée :

  1. Utilisez l'implémentation du lecteur intégré In-Page pour tester les fonctionnalités de votre lecteur, de votre plugin et de votre CSS (si CSS est nécessaire)
  2. Placez le JavaScript et le CSS du plugin dans des fichiers séparés pour des tests locaux
  3. Déployez le code du plugin et le CSS sur votre serveur une fois que vous avez corrigé des erreurs.
  4. Utilisez Studio pour ajouter le plugin et le CSS à votre lecteur
  5. Remplacez l'implémentation du lecteur intégré dans la page si vous déterminez que l'implémentation iframe est mieux adaptée (détaillée dans la section suivante)

Pour plus de détails sur ces étapes, consultez la section étape par étape : Guide de développement de plug-ins.

Intégrer iFrame ou In-Page

Lorsque vous développez des améliorations pour Brightcove Player, vous devrez décider si le code convient le mieux à l'implémentation intégrée iframe ou In-Page. La recommandation des meilleures pratiques consiste à créer un plugin à utiliser avec une implémentation iframe. Les avantages de l'utilisation du lecteur iframe sont les suivants :

  • Aucune collision avec JavaScript et/ou CSS existants
  • Réactif automatiquement
  • L'iframe facilite l'utilisation dans les applications de médias sociaux (ou lorsque la vidéo devra « voyager » dans d'autres applications)

Bien que l'intégration du lecteur d'intégration dans la page puisse être plus complexe, il y a des moments où vous planifiez votre code autour de cette implémentation. Pour généraliser, cette approche est préférable lorsque la page contenante doit communiquer avec le lecteur. En particulier, voici quelques exemples :

  • Le code dans la page contenant doit écouter et agir sur les événements du joueur
  • Le lecteur utilise des styles de la page contenante
  • L'iframe entraînera l'échec de la logique de l'application, comme une redirection à partir de la page contenant

Même si votre implémentation finale n'utilise pas le code iframe, vous pouvez toujours utiliser le code intégré In-Page avec un plugin pour votre JavaScript et un fichier séparé pour votre CSS. Cela encapsule votre logique afin que vous puissiez facilement l'utiliser dans plusieurs joueurs.

Implémentation de fonctions d'intergiciel avec Brightcove Player

Le moyen le plus simple d'utiliser une fonction intergiciel consiste à :

  1. Écrivez la fonction intergiciel.
  2. Placez la fonction intergiciel soit sur la page avec le lecteur dans une script balise, soit inclure la fonction intergiciel à l'aide de l' src attribut avec un script étiquette. Les deux implémentations seront présentées dans ce document.
  3. Enregistrez la fonction intergiciel auprès du lecteur à l'aide de la use() méthode, comme indiqué ici : 
    videojs.use('mimeType', functionName);

    Pour le type MIME, vous pouvez utiliser un type MIME vidéo standard, comme vidéo/mp4, ou utiliser un * si vous souhaitez que la fonction intergiciel agisse sur tous les mimétypes.

    Les plugins et middleware peuvent être configurés pour s'enregistrer automatiquement lorsque leurs scripts sont inclus. Dans ce cas, vous n'auriez pas besoin d'utiliser le videojs.use() méthode.

Flux d'application

La logique de base derrière cette application est :

  • Définir les fonctions de middleware
  • Enregistrez l'intergiciel auprès du lecteur
  • Définir les options de taux de lecture
  • Initialisation du bouton de fréquence de lecture

Définir les fonctions middleware

Trouvez le code qui est étiqueté :

// +++ Define the middleware functions +++

Créez les fonctions middleware suivantes:

  1. Créez une fonction middleware de prise en charge qui recalculera les heures de début et de fin de lecture en fonction de la vitesse de lecture sélectionnée.

  2. Créez une fonction middleware qui mettra à jour l'heure de début, la durée et même l'heure actuelle lors de la recherche.

    La syntaxe de base pour créer une fonction middleware est la suivante :

    const playbackrateAdjuster = function(player) {
   return {
   ...
 };

    Notez que l' player objet est passé à la fonction.

    Cette fonction middleware effectue les opérations suivantes:

    1. Écoutez l'événement Ratechange: Lorsque la vitesse de lecture est modifiée, déclenchez durationchange et timeupdate événements.

    2. Implémenter la méthode setSource (): C'est une méthode obligatoire pour chaque fonction middleware. En utilisant la syntaxe affichée signifie que le middleware sera toujours sélectionné, quel que soit le type de vidéo : 
      setSource(srcObj, next) {
  next(null, srcObj);
},

    3. Remplacer les méthodes utilisant la vitesse de lecture actuelle: Remplacez les méthodes pour duration , currentTime , buffered , seekable et played avec des valeurs calculées en utilisant la vitesse de lecture actuelle.

Enregistrer le middleware auprès du lecteur

Trouvez le code qui est étiqueté :

// +++ Register the middleware with the player +++

Enregistrez la fonction middleware auprès du lecteur à l'aide du use() méthode.

Code dans un fichier JavaScript séparé

Dans le code suivant, le JavaScript des fonctions middleware et le use() méthode pour l'enregistrer sont simplement inclus dans la page.

<video-js id="myPlayerID"
	  data-video-id="5604123528001"
	  data-account="1752604059001"
	  data-player="default"
	  data-embed="default"
	  data-application-id
	  class="video-js"
	  width="640" height="360"
	  controls muted></video>
	<script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>

  <!-- Include the rate adjuster middleware functions -->
  <script src="rate-adjuster.js"></script>

  <script type="text/javascript">
  	videojs.getPlayer("myPlayerID").ready(function(){
  		// When the player is ready, get a reference to it
  		var myPlayer = this;
  		console.log("myPlayer",myPlayer)
  		// Define the playback rate options
  		options = {"playbackRates":[0.5, 1, 1.5, 2, 4]};
  		// Initialize the playback rate button
  		myPlayer.controlBar.playbackRateMenuButton = myPlayer.controlBar.addChild('PlaybackRateMenuButton', {
      	playbackRates: options.playbackRates
  	  });
    });
  </script>

Cet exemple suppose que vous hébergerez le fichier JavaScript sur votre propre serveur. Pour le code complet, consultez le section plugin dans le dépôt GitHub.
