Exemples de codes

Les exemples de code suivants illustrent les personnalisations et les améliorations apportées à Brightcove Player. Les échantillons marqués d'un double astérisque** dépendent partiellement ou entièrement des fonctionnalités de Video Cloud. Vous pouvez également trouver une liste alphabétique d'échantillons ici.
 

Lecteur externe (publicité uniquement)

Cet exemple fournit des fonctions d'aide JavaScript qui montrent comment ajuster dynamiquement la taille d'un <div> dans le DOM de la page, y injecter le Brightcove Player, charger et lire une annonce, puis supprimer le lecteur

Scrolling Player to View with Ads

Dans le lecteur, notez que la barre de contrôle du volume horizontale remplace la barre de contrôle verticale par défaut.

Contrôle du volume horizontal (lecteur 1.x uniquement)

Dans le lecteur, une barre de contrôle de volume horizontale remplace la barre de contrôle verticale par défaut.

Volume vertical

Réglez le contrôle du volume pour afficher un curseur vertical au lieu de la disposition horizontale par défaut.

Indicateurs d'annonce dans la barre de lecture

Cet exemple montre un lecteur où les trois points de repère publicitaires associés à la vidéo sont affichés dans la barre de lecture sous forme de marqueurs bleus.

Publicité avec le plugin SSAI

Cet exemple utilise des annonces IMA définies dans un fichier VMAP pour diffuser des annonces côté serveur dans le flux vidéo. Il y a une annonce pré-roll, mi-roll et post-roll. Le fichier VMAP est défini dans la configuration de l'annonce.

Minuterie de compte à rebours

Créez un compte à rebours pour une publicité à l'aide du plugin Overlay.

 

Porte d'âge avec ModalDialog

Cet exemple utilise le ModalDialog pour couvrir initialement le lecteur. Lorsque l'utilisateur confirme son âge en cliquant sur la fenêtre modale, les contrôles sont affichés et la vidéo est lue.

CTA d'affichage des points de repère **

Le joueur montre deux CTA différents affichés en fonction des points de repère. Le premier CTA est affiché à 3 secondes, pour une durée de 5 secondes. Le deuxième CTA s'affiche à 13 secondes pour une durée de 3 secondes.

Appel à l'action dynamique **

Vous pourrez voir trois techniques d'affichage d'appel à l'action différentes avec ce lecteur.

Lecture/pause vidéo depuis iFrame Parent

Cliquez sur le bouton lecture ou pause, qui fait partie de la page parent de l'iframe, pour le voir lancer la lecture / pause de la vidéo dans le lecteur iframe.

Porte de l'âge

Cet exemple affiche une superposition qui masque les commandes du lecteur et le bouton de lecture. Lorsque l'utilisateur confirme son âge en cliquant sur la superposition, les commandes s'affichent et la vidéo est lue.

Lecture automatique avec le bouton Activer le son

La vidéo se comportera différemment selon le navigateur / la plate-forme sur lequel vous exécutez la vidéo. Si la vidéo est en lecture automatique avec audio, elle le fait. Sinon, la lecture de la vidéo commence, mais un bouton de réactivation du son apparaît.

Liste de lecture aléatoire de la vidéo d'arrière-plan **

Dans le lecteur, vous verrez des vidéos sélectionnées au hasard dans une liste de lecture qui sont continuellement lues comme vidéos d'arrière-plan. Au début de chaque vidéo, le titre et la description de chaque vidéo sont affichés.

Liste de lecture vidéo d'arrière-plan **

Dans le lecteur, vous verrez qu'il y a quatre vidéos qui sont continuellement lues comme vidéos d'arrière-plan. En outre, au début de chaque vidéo, le titre et la description de chaque vidéo sont affichés en superposant la vidéo d'arrière-plan.

Vidéo d'arrière-plan pleine fenêtre **

Dans cette rubrique, vous apprendrez à utiliser une liste de lecture pour fournir des vidéos d'arrière-plan et à redimensionner automatiquement le lecteur lorsque la fenêtre du navigateur est redimensionnée afin que la vidéo remplisse entièrement la fenêtre du navigateur.

Création d'une boucle vidéo

Dans certaines situations, vous souhaiterez peut-être utiliser une vidéo en boucle. Cet exemple boucle la vidéo 3 fois puis arrête la lecture vidéo.

Désactiver le scrubber de progression

Arrêt des autres lecteurs sur la page lorsqu'une vidéo démarre

Afficher les bumpers aléatoires

Dans cet exemple, des bumpers sélectionnés au hasard sont lus avant chaque vidéo d'une liste de lecture. Dans ce cas, les bumpers sont des publicités, mais pas tirées d'un serveur publicitaire, mais plutôt des vidéos définies dans la bibliothèque Video Cloud.

Lecture à plusieurs sessions

Naviguer sur la fin vidéo

À la fin de la très courte vidéo, vous verrez une superposition apparaître avec un lien vers la page d'accueil de Brightcove. Le plugin Overlay fourni par Brightcove est utilisé, donc aucun JavaScript n'est nécessaire. CSS est utilisé pour styliser la superposition.

Défilement du lecteur dans la vue

Faites défiler le lecteur dans et hors de la vue. Lorsque le lecteur est complètement défilé vers la vue, la vidéo commence à jouer. Lorsque vous faites défiler le lecteur hors de la vue, la lecture de la vidéo s'arrête.
