Obtenir des informations

Pour utiliser cette application de rapport de configuration, vous aurez besoin des informations d'identification appropriées.

Le moyen le plus simple d'obtenir des informations d'identification dans la plupart des cas consiste à utiliser la section Authentification de l'API Studio Admin (nécessite des autorisations d'administrateur sur votre compte). Pour plus de détails, reportez-vous à Gestion des informations d'authentification API Vous obtiendrez la permission unique suivante:

Si les autorisations dont vous avez besoin ne sont pas disponibles dans Studio, ou si vous préférez les obtenir directement à partir de l'API OAuth, utilisez votre choix de Obtenir les informations d'identification du client documents énumérés ci-dessous. Quelle que soit l'option choisie, vous devrez demander les autorisations de fonctionnement correctes. Les éléments suivants peuvent être utilisés avec cURL ou Postman pour obtenir toutes les autorisations pour l'API Player Management:

  "operations": [
    "video-cloud/players/read"
  ]

S'il y a d'autres rapports de configuration que vous aimeriez voir, veuillez envoyer un e-mail au Équipe de connaissances avec ces demandes.

Notez ce qui suit :

  • Le rapport qui affiche tous les plugins chargés dans les lecteurs d'un compte ne fonctionne pas avec la version 5 et les lecteurs plus anciens.
  • Également dans le rapport sur les plugins, les plugins commençant par brightcove / sont des plugins fournis par Brightcove. Les autres sont soit des plugins personnalisés, soit des plugins fournis par Brightcove chargés manuellement via Studio ou curl.

Utilisation du CodePen

Voici quelques conseils pour utiliser efficacement le CodePen ci-dessus :

  • Basculez l'affichage réel de l'application en cliquant sur le bouton Résultat bouton.
  • Clique le HTML/CSS/JS pour afficher UN des types de code.
  • Cliquez sur Modifier sur CodePen dans le coin supérieur droit pour incorporer ce CodePen dans votre propre compte.
  • Retrouvez tout le code associé à cet exemple dans ce Dépôt GitHub.

Code proxy

Pour créer votre propre version, l'exemple d'application sur cette page, vous devez créer et héberger votre propre proxy. (Les proxy utilisés par Brightcove Learning Services n'acceptent que les demandes des domaines Brightcove.) Un exemple de proxy, très similaire à celui que nous utilisons, mais sans les contrôles qui bloquent les demandes provenant de domaines non-BrightCove, peut être trouvé dans ce dépôt GitHub. Vous y trouverez également des instructions de base pour l'utiliser, ainsi qu'un guide plus détaillé sur la création d'applications autour du proxy dans Utilisation des API REST.
