Gestion des droits numériques (DRM)

DRM (Digital Rights Management) protège votre contenu vidéo en chiffrant les données vidéo et en les déverrouillant en fonction des règles de licence. Lorsque le contenu vidéo est chargé dans un lecteur Brightcove, le lecteur rappelle un serveur de licences et obtient l'autorisation de lire la vidéo. Cette transaction est transparente pour les spectateurs. Voir le Aperçu: Gestion des droits numériques (DRM) dans Video Cloud document pour plus d'informations.

Chiffrement HLS (HLSE)

Bien qu'il ne soit pas aussi sûr que DRM, le chiffrement HLS rend la visualisation non autorisée de vos vidéos plus difficile. Voir Protection des vidéos avec le cryptage HLS pour plus de détails sur le fonctionnement de HLSe et comment activer votre compte.

Géofiltrage

Le filtrage géo-filtrant peut être utilisé pour la protection du contenu. Voir le Configuration de la disponibilité vidéo document pour obtenir des instructions sur la façon de filtrer géographiquement vos vidéos Video Cloud et Messagerie de filtrage géographique document expliquant comment gérer les erreurs de filtrage géographique dans Brightcove Player.

TTL (authentification par jeton)

Si vous fournissez du contenu premium ou confidentiel, vous chercherez tous les moyens de garantir sa sécurité. Vous ne souhaitez pas que les utilisateurs finaux puissent copier des URL vidéo ou continuer à lire du contenu sans l'autorisation appropriée. Brightcove prend en charge la signature VOD des URL HLS, MP4 et DASH. La signature est prise en charge pour les configurations Brightcove House et BYO CDN avec Akamai, Fastly, Cloudfront et JOCDN.

Par défaut, les jetons TTL ont une durée de vie très courte. Vous pouvez augmenter la durée de vie si nécessaire (pour les flux MRSS et autres cas d'utilisation) en contacter l'assistance Brightcove pour qu'ils définissent la valeur de durée de vie du jeton plus élevée. La fourchette admissible est d'une heure à un an. N'oubliez pas qu'il s'agit d'un paramètre de compte, cependant. Ne définissez pas la valeur si élevée qu'elle pourrait compromettre la sécurité de vos autres vidéos.

Les restrictions de domaine

Les restrictions de domaines peuvent uniquement être mises en place sur les lecteurs pour limiter les domaines dans lesquels les lecteurs peuvent être utilisés. Si le code de publication du lecteur est copié et utilisé sur un autre site, les restrictions de domaines empêcheront le lecteur de charger des vidéos. Les éditeurs doivent utiliser le code intégré dans la page pour réellement faire fonctionner la lecture sur les domaines qui ont été mis en liste blanche. Les éditeurs ne peuvent pas utiliser le code d'intégration iframe sur un domaine de liste blanche (ou tout autre domaine) et s'attendent à ce que la lecture fonctionne correctement.

Voir le Configuration des restrictions de contenu du lecteur document pour savoir comment limiter le domaine de votre lecteur et le Messagerie de restrictions de domaine document expliquant comment gérer les erreurs de restriction de domaine dans Brightcove Player.

Restrictions IP

Avec les restrictions IP, vous pouvez configurer les lecteurs pour autoriser les adresses IP sur la liste blanche en dehors des États-Unis d'accéder à votre contenu Video Cloud. Veuillez contacter le service à la clientèle pour obtenir de l'aide. Pour les joueurs utilisés en dehors de l'Amérique du Nord avec des restrictions IP, assurez-vous de vérifier les informations dans le Restriction de la lecture vidéo à l'aide de l'adresse IP pour éviter les problèmes qui peuvent survenir.
