Plusieurs pistes audio

{{ page.description }}

Introduction

Lecteur Brightcove prend en charge la vidéo avec plusieurs pistes audio alternatives. Si la vidéo est préparée avec plusieurs pistes audio alternatives, une icône de casque spéciale () apparaît automatiquement dans la barre de contrôle. Encore une fois, si vous avez une vidéo correctement encodée, l'icône apparaîtra, vous n'avez rien d'autre à faire. Une fois que vous avez cliqué sur l'icône, le spectateur de la vidéo peut choisir entre les pistes audio.

Remarques :
  • Les pistes audio multiples ne sont prises en charge que pour les comptes activés pour la diffusion dynamique, et le compte doit être activé spécifiquement pour plusieurs pistes audio. Contactez votre représentant pour plus d’informations.
  • Les pistes audio multiples sont prises en charge uniquement pour le contenu VOD.
  • Une fois qu'un compte est activé pour plusieurs pistes audio, toutes les sources vidéo existantes avec plusieurs pistes audio devra être re-transcodé.
  • Lorsque vous ajoutez des pistes audio supplémentaires à une vidéo, elles doivent toutes avoir le même profil bitratique/rendu pour garantir une lecture réussie dans HLS (la lecture DASH fonctionnera toujours).
  • Les pistes audio multiples ne sont actuellement pas prises en charge pour Smooth.
  • Le téléchargement d'un fichier vidéo source avec plusieurs pistes audio ne générera qu'une seule piste audio après le transcodage. Généralement, l'une des pistes est marquée par défaut, mais s'il n'y a pas de valeur par défaut, le système Ingest choisira la "meilleure" piste audio en fonction de la qualité stéréo et du débit binaire le plus élevé et la marquera comme en / main.

Exemple

Une fois que vous cliquez sur Lecture, vous verrez l'icône du casque dans la barre de contrôle à partir de laquelle vous pouvez choisir entre deux pistes audio alternatives.

Cas d'utilisation

Voici quelques cas d'utilisation courants de plusieurs pistes audio alternatives:

  • Support multilingue
  • Audio stéréo ou multicanal (son surround)
  • Piste de commentaire
  • Audio descriptif

Icône et menu de style

Si les couleurs par défaut ne répondent pas à vos besoins, vous pouvez modifier la couleur de l'icône du casque, l'option sélectionnée et d'autres options. Le tableau et la capture d'écran suivants détaillent les sélecteurs que vous devrez utiliser. (Notez que les couleurs affichées dans la capture d'écran ne sont PAS les valeurs par défaut, mais des couleurs à identifier facilement.)

Sélecteurs pour l'icône audio
Élément Sélecteur
Icône de casque (rouge sur la capture d'écran) .video-js .vjs-control-bar .vjs-audio-button
Option sélectionnée (jaune sur la capture d'écran) .video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item.vjs-selected
Option (s) disponible (s) (vert sur la capture d'écran) .video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item

Ce qui suit est le réel CSS qui a affecté les changements dans la capture d'écran :

.video-js .vjs-control-bar .vjs-audio-button {
  color: red;
}
.video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item.vjs-selected {
  background-color: yellow;
}
.video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item {
  background-color: green;
}

Problèmes connus

  • Si vous ajoutez un rendu distant HLS (m3u8) via un <source> dans le lecteur, les spectateurs ne pourront pas sélectionner plusieurs pistes audio dans Chrome sur le système d'exploitation Android.
