v2.19.1

Remarque : Cette version inclut des correctifs pour prendre en charge la lecture automatique des annonces HMTL5 sur les appareils iOS 10 et Android.

  • Lire automatiquement des annonces HTML5 dans des lecteurs qui ne sont pas désactivés et / ou ne comprennent pas playsinline jouera correctement lorsque l'utilisateur lance la lecture en appuyant ou en cliquant sur les appareils iOS 10 et Android
  • Lire automatiquement des annonces HTML5 dans les lecteurs qui incluent muted ET playsinline sera correctement lu automatiquement sur les appareils iOS 10 et Android
  • Corrige le bouton Mute sur les annonces HTML5 iOS 10 et Android
    • Quand le muted L'attribut est présent, le bouton de désactivation s'affichera pour permettre à l'utilisateur de réactiver l'annonce
    • Le bouton de sourdine ne s'affiche plus sur les périphériques qui ne prennent pas en charge le contrôle du volume par programmation
    • Le bouton de sourdine désactivera correctement l'annonce lorsqu'il est tapé ou cliqué.
    • Le paramètre de désaction/désinactivité persiste de l'annonce vers le contenu principal

En outre, le SDK HTML5 IMA a publié une fonctionnalité pour désactiver la lecture personnalisée sur les appareils iOS 10+. Cette fonctionnalité détermine si la lecture en ligne ou la lecture native des annonces est utilisée, y compris l'ajout de la prise en charge des annonces ignorables TrueView. Pour plus d'informations sur ce paramètre, consultez le Documentation du SDK IMA HTML5.

Des informations sur la façon d'appliquer ce paramètre à l'aide du plug-in Brightcove IMA SDK peuvent être trouvées dans notre Publicité avec le plugin IMA3 documentation sous ima3SdkSettings.

v2.19.0

La version 2.19.0 pouvait entraîner l'échec de certaines implémentations programmatiques du plug-in IMA3. Veuillez consulter le Implémenter en utilisant du code dans ce document pour vous assurer que votre code respecte les meilleures pratiques recommandées. L'erreur suivante indique quand vous devez mettre à jour votre code actuel :

VIDEOJS: ERREUR : videojs-contrib-ads n'a pas vu d'événement loadstart 5 secondes après avoir été initialisé, mais une source est présente. Cela indique que videojs-contrib-ads a été initialisé trop tard. Il doit être initialisé immédiatement après video.js dans la même coche. Par conséquent, certaines publicités ne seront pas diffusées et certains événements multimédias seront incorrects.

Pour plus d'informations, voir cet article dans le Brightcove Web Player Developer Forum. Pour obtenir des informations générales sur le test des lecteurs et des plugins ensemble, consultez le Test de la version du lecteur / plug-in sujet.

2 nov. 2016

v2.15.0

  • Nouvelle fonctionnalité de variable de page dans les macros d'annonces
    • Dans IMA, les macros publicitaires sont prises en charge dans l'URL du serveur publicitaire
    • Une URL avec une macro de variable de page peut ressembler à ceci: http://example.com?foo={pageVariable.foo} (cette macro serait remplacée par la valeur de la variable foo dans le contexte JavaScript du navigateur lors de la demande d'annonce)
    • Lorsque votre demande d'annonce est faite si la variable de page n'est pas encore disponible, votre macro ne fonctionnera pas
    • Vous pouvez en savoir plus sur la nouvelle fonctionnalité de macro d'annonce ici: https://github.com/videojs/videojs-contrib-ads#macros
  • Maintenant, utilisez try/catch lors de la destruction du gestionnaire d'annonces au cas où il ne serait pas encore créé
    • Ceci est pour plus de sécurité dans les cas de bord, vous ne devriez pas vous attendre à voir de différence.
  • Temporairement ré-ajouté contentplayback un événement
    • Cet événement faisait partie de l'intégration entre le plugin Brightcove IMA et l'open source videojs-contrib-annonces projet; Brightcove prévoit de le supprimer à l'avenir
