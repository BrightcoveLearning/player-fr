    v2.19.1

    Remarque : Cette version inclut des correctifs pour prendre en charge la lecture automatique des annonces HMTL5 sur les appareils iOS 10 et Android.

    • Lire automatiquement des annonces HTML5 dans les lecteurs qui incluent muted ET playsinline sera correctement lu automatiquement sur les appareils iOS 10 et Android

Cette fonctionnalité détermine si la lecture en ligne ou la lecture native des annonces est utilisée, y compris l'ajout de la prise en charge des annonces ignorables TrueView. Pour plus d'informations sur ce paramètre, consultez le Documentation du SDK IMA HTML5.

    Des informations sur la façon d'appliquer ce paramètre à l'aide du plug-in Brightcove IMA SDK peuvent être trouvées dans notre Publicité avec le plugin IMA3 documentation sous ima3SdkSettings.

    v2.19.0

    La version 2.19.0 pourrait entraîner l'échec de certaines implémentations programmatiques du plugin IMA3. Veuillez consulter le Implémenter en utilisant du code dans ce document pour vous assurer que votre code respecte les meilleures pratiques recommandées. L'erreur suivante indique quand vous devez mettre à jour votre code actuel :

    VIDEOJS: ERREUR : videojs-contrib-ads n'a pas vu d'événement loadstart 5 secondes après avoir été initialisé, mais une source est présente. Cela indique que videojs-contrib-ads a été initialisé trop tard. Il doit être initialisé immédiatement après video.js dans la même coche. Par conséquent, certaines publicités ne seront pas lues et certains événements multimédias seront incorrects.

    Pour plus d'informations, voir cet article dans le Brightcove Web Player Developer Forum. Pour obtenir des informations générales sur le test des lecteurs et des plugins ensemble, consultez le Test de la version du lecteur / plug-in sujet.

    • Les paramètres de volume persistent lorsque vous ignorez les annonces

      • Meilleur nettoyage lorsque le joueur est éliminé

    • Nouvelle fonctionnalité de variable de page dans les macros d'annonces

      • Une URL avec une macro de variable de page peut ressembler à ceci: http://example.com?foo={pageVariable.foo} (cette macro serait remplacée par la valeur de la variable foo dans le contexte JavaScript du navigateur lors de la demande d'annonce)
