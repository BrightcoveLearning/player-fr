Journal des modifications du plug-in Picture-in-Picture

Les modifications apportées au plugin sont les suivantes :

Changelog

21 avr. 2020

v1.1.3

  • Résolution d'un problème où la superposition PiP ne fonctionnait pas après la sortie du mode plein écran

29 janv. 2020

v1.1.2

  • Correction d'un problème dans lequel le piP dans la page pouvait être bloqué dans une boucle qui provoquait le scintillement du joueur.

3 déc. 2019

v1.1.1

  • Correction d'un problème qui faisait que l'image dans la page pouvait être bloquée dans une boucle, ce qui provoquait le scintillement du joueur

20 févr. 2019

v1.1.0

  • Ajoutez la prise en charge du futur code d'intégration réactif basé sur Studio. Des améliorations sont à venir sur les codes d'intégration réactifs fournis par Studio. Ces modifications garantiront que le plug-in PIP prend en charge correctement ces codes d'intégration lorsque ces modifications sont effectuées.

4 déc. 2018

v1.0.4

  • Correction d'un bug : Ne pas inclure loscore dans les lots
  • Correction d'un bug : Correction d'un problème où AllowOnMobile n'était pas respecté
  • Correction d'un bug : Supprimer le script postinstall pour éviter les problèmes d'installation
  • Mise à jour à l'aide du générateur de plugin v7.2.0

3 août 2018

v1.0.3

  • Correction de bug : Définir l'option affichable lors de l'instanciation du plugin

11 juin 2018

v1.0.2

  • Correction d'une condition de course rare où un joueur PIP qui a été initialisé tardivement pouvait déclencher le comportement PIP lors de l'initialisation ; maintenant, il attend l'activité de l'utilisateur avant d'autoriser le mode PIP à activer

25 mai 2018

v1.0.1

  • Correction du mode Image dans l'image pour les joueurs réactifs

26 mars 2018

v1.0.0

  • Permettez au lecteur de passer en mode image dans l'image où il est détaché de la page et fixé à la fenêtre d'affichage.
