Options disponibles pour le code d'intégration standard (iframe)

{{ page.description }}

Introduction

le iframe Le code d'implémentation du lecteur offre la facilité d'utilisation du lecteur en tant qu'unité unique et contenue. Cela est souvent nécessaire pour les applications et le partage de médias sociaux.

Utilisation

Remarque : Si vous êtes un client Brightcove Player, vous n'utiliserez pas le videoId paramètre de requête pour attribuer une vidéo à votre lecteur. Vous devez soit attribuer la vidéo dans le Joueurs module ou en utilisant curl.

Pour utiliser le iframe code suivez ces étapes:

  1. Utilisez le module PLAYERS pour créer un lecteur.
  2. Accédez au module MEDIA et publiez une vidéo à l'aide du lecteur nouvellement créé.
  3. Copiez le la norme code intégré. Le code HTML sera similaire à ce qui suit : 
          <iframe src="https://players.brightcove.net/921483702001/default_default/index.html?videoId=4570583963001"
        allowfullscreen
        webkitallowfullscreen
        mozallowfullscreen></iframe>
  4. Copiez le code HTML à partir du navigateur et collez-le dans une page HTML complète. body
  5. Parcourez la page HTML pour voir le lecteur fonctionne.

Paramètres de requête

De nombreux paramètres de requête sont disponibles pour le iframe tag pour fournir des informations supplémentaires sur la manière dont un joueur doit se comporter. Le tableau ci-dessous détaille les paramètres disponibles.

Lors de l'utilisation de paramètres de requête qui ne peuvent être que true ou false (type de données Booléen), il vous suffit d'inclure le paramètre de requête pour l'activer ou de l'exclure pour le désactiver. Un exemple est montré ici en utilisant muted.

<iframe src="https://players...index.html?videoId=4784463159001&muted"

Si la valeur booléenne autoplay est définie, le comportement de lecture automatique natif du navigateur est utilisé. S'il est défini sur l'une des trois valeurs de chaîne valides suivantes, ce qui suit se produit :
  • play: Le joueur essaiera manuellement d'appeler la lecture.
  • muted: Le joueur éteindra manuellement le joueur et appelle la lecture.
  • any: Le joueur essaiera d'abord d'appeler le jeu. Si cela échoue, coupez le lecteur et appelez la lecture.
Si l'un des éléments ci-dessus échoue, le joueur affiche le « Big Play Button » comme si la lecture automatique était définie sur false.

détails de préchargement

Le paramètre de préchargement indique au navigateur si le téléchargement des données vidéo doit commencer ou non dès que la balise vidéo est chargée. Les options sont auto, métadonnées et aucun. Pour la technologie de lecture HLS, les paramètres signifient des choses légèrement différentes et notées ci-dessous.

  • auto (défaut): Commencez à charger la vidéo immédiatement (si le navigateur est d'accord). Certains appareils mobiles tels que les iPhones et les iPad ne préchargeront pas la vidéo afin de protéger la' bande passante de leurs utilisateurs. C'est pourquoi la valeur est appelée auto et non pas quelque chose de plus final comme vrai.

    Pour la technologie de lecture HLS, l'option automatique signifie que le lecteur télécharge le fichier manifeste m3u8 et seulement 1 segment au lieu de 3-4, comme c'était le cas avant la version 5.3 de Brightcove Player.

  • métadonnées: Chargez uniquement les métadonnées de la vidéo, qui incluent des informations telles que la durée et les dimensions de la vidéo.

    Pour la technologie de lecture HLS, les métadonnées sont identiques à celles de l' auto.

  • aucun: Ne préchargez aucune des données vidéo. Cela attendra jusqu'à ce que l'utilisateur clique sur la lecture pour commencer le téléchargement. Notez que sur iOS, il peut y avoir un téléchargement de segments même avec ce paramètre. Voir le Lecteur Brightcove et iOS document pour plus d'informations.

    Pour la technologie de lecture HLS, rien signifie que le joueur ne chargera plus rien, y compris le manifeste m3u8, jusqu'à ce que l'utilisateur appuie sur play.
      <video-js preload ...>
      or
      { "preload": "auto" }
Remarque : Lorsque vous utilisez SSAI (Server-Side Ad Insertion), Brightcove recommande de paramétrer preload=none. S'il preload est défini sur auto ou metadata, les premières images de la vidéo le flux sera demandé, ce qui compte l'annonce comme diffusée même si l'utilisateur ne démarre pas la lecture.
