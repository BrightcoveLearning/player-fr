    Notes de mise à jour du lecteur Brightcove

    Cette page suit les informations de publication de Brightcove Player

    Note : Par défaut, les mises à jour sont appliquées automatiquement à tous les lecteurs. Pour empêcher la mise à jour automatique d'un lecteur, vous devez le définir sur la mise à jour manuelle (également connue sous le nom de verrouillage d'un lecteur). Vous pouvez le faire dans le module Studio Players (site externe) ou à l'aide de l' API Player Management.

    La désactivation des mises à jour automatiques doit être une mesure temporaire, appliquée lorsque vous résolvez les problèmes créés par la mise à jour. Si vous ne mettez pas à jour les joueurs vers la version actuelle, vous manquez de nouvelles fonctionnalités et corrections de bogues, et les anciennes versions du lecteur cesseront d'être prises en charge à un moment donné.

    Également , vous pouvez voir les versions de Brightcove Player et les versions de plug-in dans le Annonces de sortie de Brightcove Player Groupe Google.

    • Correction d'échecs avec des aperçus et des URL de fichiers avec des paramètres de requête

    VERSION 5 - 5.28.1

    Brightcove Player version 5 n'est plus pris en charge. Les lecteurs publiés continueront de fonctionner et ne seront pas désactivés, mais Brightcove ne peut garantir la compatibilité future avec les navigateurs, les API, les formats vidéo, les intégrations et d'autres changements pouvant survenir en dehors du lecteur.

    Notre recommandation est de passer à la version la plus récente de la version 6. Si vous avez besoin d'aide pour ce faire, veuillez contacter l'assistance Brightcove.

    1 Il est à noter que, dans ce contexte, rendre le lecteur "réactif à la taille du navigateur par défaut" est différent de rendre le lecteur fluide. Ce qui suit devrait clarifier la nouvelle fonctionnalité:

    • Pour être clair: les joueurs statiques ne deviendront pas soudainement fluides!
    • L'utilisation de «responsive» dans ce contexte signifie que nous avons activé la nouvelle option responsive Video.js dans Brightcove Player par défaut (alors qu'elle est désactivée par défaut dans Video.js). Cela signifie que la «réactivité» n'est pas réellement basée sur la taille de la fenêtre du navigateur, mais sur la taille du lecteur intégré.
    • Deux petits ajustements de l'interface utilisateur ont été effectués sur la base de cette nouvelle option Video.js. Il est prévu que ces ajustements ne devraient pas affecter de nombreux utilisateurs, le cas échéant, mais dans un souci d'exhaustivité, ils concernent:
      • Boîte de dialogue des paramètres des sous-titres
      • Boîte de dialogue du plugin social
    • Les utilisateurs qui ont des intégrations réactives depuis le Studio ou via l'option fluide de Video.js pourront profiter de la nouvelle vjs-layout-* classes pour améliorer les personnalisations de leur interface utilisateur en fonction de la taille du joueur. Et les utilisateurs qui ont des intégrations de taille statique bénéficieront toujours d'un vjs-layout-* classe appliquée à leur lecteur, mais elle ne changera que si la taille du lecteur change lors de l'exécution.
    • Pour les implémentations de lecteurs de petite largeur (comme 425px), il est possible que certains boutons de la barre de contrôle soient masqués. Bien que vous perdiez les fonctionnalités réactives, le code suivant empêchera les boutons d'être masqués:

    videojs.getPlayer('myPlayerID').ready(function() {

    Brightcove Player version 5 prend en charge la vidéo HLS sans utiliser Flash.
