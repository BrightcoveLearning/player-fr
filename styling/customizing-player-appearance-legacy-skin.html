Personnalisation de l'apparence du joueur - Skin hérité

Ce document contient des informations sur l'habillage Brightcove Player 1.x (graphite), et non sur l'habillage le plus récent. Cette transition vers la nouvelle peau a eu lieu la deuxième semaine de décembre 2015. Pour la version la plus récente du document, consultez ce document.

Cette rubrique est une introduction à la personnalisation de l'apparence du nouveau lecteur / styling / customizing-player-looks.html. Le contenu utilise les outils de développement de Chrome, mais tous les navigateurs modernes ont des fonctionnalités équivalentes. Pour une brève introduction à l'utilisation des outils de développement dans différents navigateurs, consultez le Bases du débogage JavaScript document.

Créer sa propre peau

Si vous ne souhaitez pas utiliser l'habillage par défaut et créer le vôtre, cela est possible à l'aide de l'API Player Management. Le Guide de configuration du joueur fournit les informations nécessaires.

Lecteur

Remarque : Ce document suppose une certaine connaissance du CSS et utilise les outils de développement d'un navigateur pour visualiser les classes des éléments HTML et examiner les CSS applicables. Si vous ne possédez pas les connaissances de base requises, il serait préférable que vous Démarrage rapide: Personnalisation du joueur - Legacy Skin.

Vous pouvez modifier l'apparence du joueur, mais vous devez d'abord avoir un moyen d'y remédier. Pour voir cela, parcourez une page HTML avec le code du lecteur embed_in_page inséré. dans le Éléments section des outils de développement localiser le premier <div> et vous verrez qu'il a une classe avec la valeur video-js assigné, entre autres.

Une fois que vous le savez, vous pouvez utiliser des styles pour modifier le lecteur lui-même. Par exemple, pour changer la taille du lecteur, vous pouvez utiliser ce style:

<style>
  .video-js {
    width: 640px;
    height: 360px;
  }
</style>

lecteur iframe

Si vous utilisez l'implémentation iframe du lecteur, la situation est différente. Vous verrez toujours le joueur avec le video-js class, mais ce sera bien sûr à l'intérieur de l'iframe.

Les styles que vous créez continueront de fonctionner avec le lecteur dans l'iframe, mais un fichier CSS devra être créé et associé au lecteur utilisant Studio. Vous faites cela en vous rendant au Joueurs module, puis cliquez sur la vidéo à laquelle vous souhaitez associer le CSS, puis dans le Plugins section ajoutez le chemin de votre fichier CSS.

Si vous souhaitez personnaliser l'iframe lui-même, vous pouvez le faire en ajoutant un class attribut au iframe élément (ligne 27). Dans ce cas, la valeur de la classe est videoFrame. Ensuite, utilisez un <style> balise à modifier à volonté. Dans l'exemple ci-dessous, le width et height sont modifiés.

<!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Customize iframe</title>

  <!-- Page styles -->
  <style>
    .videoFrame {
      width: 640px;
      height: 360px;
    }
  </style>
</head>

<body>
  <iframe class="videoFrame" src='//players.brightcove.net/1752604059001/68f30408-8fb4-431d-accb-8c5baa8c4790_default/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>
</body>

</html>

Bouton jouer

Si vous souhaitez modifier l'apparence du bouton de lecture, vous devez d'abord savoir comment y remédier. Dans Chrome, assurez-vous que Outils de développement sont ouverts puis faites un clic droit sur le bouton. Parmi les sélections qui apparaissent, choisissez Inspecter l'élément.

dans le Éléments section des outils de développement, vous verrez le code HTML qui correspond à l'élément bouton.

À partir du code surligné ci-dessus, vous pouvez voir la classe attribuée au bouton est vjs-big-play-button. La couleur et l'opacité du bouton de lecture (pour correspondre au tigre) seront modifiées avec ces styles:

  <style>
    .vjs-big-play-button {
      background-color: #B37D5B;
      opacity: .6;
    }
    .video-js {
      width: 640px;
      height: 360px;
    }
  </style>

On pourrait penser que cette approche fonctionnerait, mais ce n'est pas le cas. Lorsque vous travaillez avec CSS, vous devez être conscient de la spécificité des propriétés. Cela fait référence à la façon dont le navigateur détermine les valeurs de propriété à appliquer à un élément en fonction de leur pertinence.

Une façon d'appliquer les modifications du bouton consiste à référencer un identifiant dans le sélecteur CSS. C'est parce que les identifiants ont une spécificité CSS élevée. Pour plus de détails, consultez le Spécificité CSS section

Tout d'abord, ajoutez un id attribut au div élément pour le code du joueur.

    <video-js id="video_1"
      data-account="1752604059001"
      data-player="68f30408-8fb4-431d-accb-8c5baa8c4790"
      data-embed="default"
      class="video-js" controls></video-js>

    <script src="//players.brightcove.net/1752604059001/68f30408-8fb4-431d-accb-8c5baa8c4790_default/index.min.js"></script>

Ensuite, référencez le gros bouton de lecture du lecteur dans le CSS comme suit:

  <style>
    #video_1 .vjs-big-play-button {
      background-color: #B37D5B;
      opacity: .6;
    }
    .video-js {
      width: 640px;
      height: 360px;
    }
  </style>

Voici le résultat du changement du bouton du gris foncé à l'orange :

display: none;

Progrès et volume

La dernière personnalisation de l'apparence que nous examinerons dans ce document sera de changer la couleur des indicateurs de progression et de volume. La couleur par défaut pour les deux est le fuchsia, comme indiqué ici:

Le défi consiste ici à trouver les noms de classes pour les éléments. En utilisant la même technique que ci-dessus, un clic droit sur un élément en choisissant Inspecter l'élément , vous pouvez accéder à l'ensemble de l'indicateur de progression, puis en explorant vers le bas, vous verrez le vjs-play-progress classe.

Dans le Chrome Éléments panneau, passez la souris sur le div élément avec un nom de classe de vjs-play-progress pour voir ce qui suit dans le lecteur:

Un outil dans Chrome est utile lors de la vérification de l'élément de niveau de volume. Notez que vous ne voyez le niveau de volume que lorsque vous passez la souris sur le bouton de volume. Pour afficher le niveau de volume lors de l'inspection et de la recherche du bon élément, procédez comme suit:

  • Faites un clic droit sur le bouton de volume et inspectez-le
  • Clique le Bouton de bascule d'élément (surligné en bleu ci-dessous) en haut à droite du modes section
  • Vérifier la :flotter (surligné en rouge dans le modes section) pour simuler le survol de la souris pendant que vous inspectez les éléments
  • Explorez jusqu'à ce que vous trouviez l'élément de niveau de volume (surligné en rouge dans le Éléments section)

Maintenant le background-color de ces deux éléments doit être stylisé. Vous voulez qu'ils aient la même couleur que le bouton, alors modifiez vos styles comme indiqué ici:

  <style>
    #video_1 .vjs-big-play-button {
      opacity: .6;
    }
    #video_1 .vjs-big-play-button,
    #video_1 .vjs-play-progress,
    #video_1 .vjs-volume-level {
      background-color: #B37D5B;
    }
    .video-js {
      width: 640px;
      height: 360px;
    }
  </style>

Notez que nous préfixons chaque classe avec l'identifiant du joueur div élément. Cela fournit de la pertinence pour les changements de classe en ce qui concerne la spécificité CSS. Cela sera discuté dans la section suivante.

Contrôle la visibilité

Vous pouvez dicter la durée de visibilité des contrôles. Vous utilisez le options().inactivityTimeout propriété de faire cela. Vous définissez la valeur égale à un nombre qui correspond aux secondes pendant lesquelles les commandes restent visibles après la fin de toute activité sur le lecteur. Pour que les contrôles restent visibles tout le temps, définissez la valeur sur 0.

Le code suivant montre le réglage de la inactivityTimeout à 0 , ce qui rendra les contrôles toujours visibles.

<video-js id="myPlayerID"
  data-account="1752604059001"
  data-player="04ee9f58-9d08-43ea-803f-abda86776db2"
  data-embed="default"
  class="video-js" controls></video-js>
<script src="//players.brightcove.net/1752604059001/04ee9f58-9d08-43ea-803f-abda86776db2_default/index.min.js"></script>

<script>
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this;
    myPlayer.options().inactivityTimeout = 0;
  });
</script>

Ordre des icônes de la barre de contrôle

Vous pouvez contrôler l'ordre des icônes dans la barre de contrôle en utilisant le order style a attribué un numéro. Par exemple, 5 placera une icône après la barre de progression mais avant le bouton de volume, et 9 placera l'icône à l'extrême droite de la barre de contrôle. Pour les navigateurs plus anciens, d'autres styles peuvent être nécessaires, comme indiqué ici:

{
  order: 5; /* after the progress bar but before the volume button */
  -webkit-box-ordinal-group: 5;
  -moz-box-ordinal-group: 5;
  -webkit-order: 5;
  -ms-flex-order: 5;
}

Un exemple d'ajout d'une icône (pour télécharger un fichier dans ce cas) est présenté dans le Exemple de lecteur Brightcove: Télécharger le document Video Plugin .

Spécificité CSS

La spécificité CSS fait référence à la façon dont les navigateurs déterminent quelles valeurs de propriétés sont appliquées à un élément en fonction de leur pertinence. Pour plus de détails sur ce concept, consultez le Spécificité article.

Dans Chrome, inspectez le gros bouton de lecture. Notez que la background propriété est définie à l'aide de plusieurs classes. Cela signifie que nous ne pouvons pas utiliser uniquement la .vjs-big-play-button classe unique pour remplacer cette propriété.

Voici trois approches pour personnaliser le gros bouton de lecture :

  • Référence à un identifiant attribué au joueur.
  • Utilisez plusieurs classes pour augmenter la pertinence des propriétés.
  • Utilisez JavaScript pour définir l'attribut id sur le lecteur, puis le référencer dans le CSS.

Référencement d'un identifiant

Étant donné qu'un id attribut présente une grande pertinence en ce qui concerne la spécificité CSS, l'ajout d'un attribut id au sélecteur CSS remplacera la valeur background-color propriété.

Si vous ne l'avez pas déjà fait, ajoutez un id attribut au video élément comme suit:

    <video-js id="video_1"
      data-account="1752604059001"
      data-player="68f30408-8fb4-431d-accb-8c5baa8c4790"
      data-embed="default"
      class="video-js" controls></video-js>

Dans le CSS, ajoutez la valeur id à la .vjs-big-play-button sélecteur de classe. Remplacez la couleur et l'opacité d'arrière-plan comme suit:

  <style>
    #video_1 .vjs-big-play-button {
      background-color: #B37D5B;
      opacity: .6;
    }
    .video-js {
      width: 640px;
      height: 360px;
    }
  </style>

Vous devriez voir que le gros bouton de lecture est maintenant orange.

Utilisation de plusieurs classes

Depuis le background La propriété du gros bouton de lecture est définie à l'aide de deux classes, vous aurez besoin d'au moins deux classes dans le sélecteur CSS pour remplacer cette propriété.

Ajouter le .video-js classe à la .vjs-big-play-button sélecteur de classe. N'oubliez pas que le .video-js la classe est définie sur le conteneur vidéo.

Avec ces valeurs, nous disons de sélectionner tous les éléments avec un nom de classe de vjs-big-play-button qui sont des décendants d'éléments avec un nom de classe de .video-js.

<style>
  .video-js .vjs-big-play-button {
    background-color: #B37D5B;
    opacity: .6;
  }
  .video-js {
    width: 640px;
    height: 360px;
  }
</style>

Vous devriez voir que le gros bouton de lecture est maintenant orange. L'utilisation de plusieurs classes peut être délicate et peut affecter plusieurs éléments, alors qu'un identifiant est spécifique à un élément.

Utilisation de JavaScript

Vous pouvez également utiliser JavaScript pour ajouter dynamiquement un identifiant au gros bouton de lecture. Ensuite, vous pouvez référencer cet identifiant à partir du CSS.

Ajoutez du code pour ajouter un identifiant à l'élément gros bouton de lecture.

  • Ligne 173 : Obtient une référence au premier élément ayant une classe de vjs-big-play-button.
  • Ligne 174 : Ajoute un id attribut au grand bouton de lecture.
  <script type="text/JavaScript">

    var playButton = document.getElementsByClassName("vjs-big-play-button")[0];
    playButton.setAttribute("id", "myPlayButton");

  </script>

Dans le CSS, modifiez le sélecteur pour utiliser le nom myPlayButton d'identifiant.

  <style>
    #myPlayButton {
      background-color: #B37D5B;
      opacity: .6;
    }
    .video-js {
      width: 640px;
      height: 360px;
    }
  </style>

Vous devriez voir que le gros bouton de lecture est maintenant orange.
