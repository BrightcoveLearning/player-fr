    Brève introduction : Personnalisation du joueur - Skin hérité

    Ce document contient des informations sur l'habillage Brightcove Player 1.x (graphite), et non sur l'habillage le plus récent. Cette transition vers la nouvelle peau a eu lieu la deuxième semaine de décembre 2015. Pour la version la plus récente du document, voir ce document.

    Dans cette rubrique, des étapes pratiques sont fournies pour personnaliser l'apparence du lecteur Brightcove lorsqu'il est suivi. Pour personnaliser l'apparence du lecteur Brightcove, vous devez utiliser les feuilles de style en cascade (CSS). Ce document passera par le processus de modification de l'apparence et de l'emplacement du bouton de lecture qui superpose initialement la vidéo.

    Présentation

    Ce document aura plusieurs sections comme l'utilisation de CSS avec l'intégration dans la page et les implémentations iframe du lecteur sont assez différentes.

    Le contenu de ce document utilise les outils de développement de Chrome, mais tous les navigateurs modernes ont des fonctionnalités équivalentes. Pour une brève introduction à l'utilisation des outils de développement dans différents navigateurs, consultez le document Debugging Basics .

    Remarque : Un style de lecteur de base est fourni à l'intérieur de Video Cloud Studio. Pour plus d'informations, consultez la rubrique Personnalisation du style des lecteurs.

    Déterminer le sélecteur de joueur

    Les sélecteurs CSS vous permettent de sélectionner et de manipuler des éléments HTML sur une page. Les sélecteurs CSS sont utilisés pour « trouver » (ou sélectionner) des éléments HTML en fonction de leur id, classe, type, attribut, et plus encore. La première chose que vous devez faire lorsque vous essayez de modifier l'apparence d'un élément sur une page HTML est de déterminer le sélecteur à utiliser pour aborder l'élément. Les étapes suivantes montrent comment trouver le sélecteur pour le joueur, puis modifier la taille du joueur.

    1. Dans Video Cloud Studio, ouvrez le module Media et sélectionnez une vidéo à publier.
    2. Copiez le En page code de publication.
    3. À l'aide de votre éditeur HTML, créez une nouvelle page HTML.
    4. Collez le code de publication dans le corps de votre page HTML.
    5. Parcourez la page nouvellement créée. Par défaut, le lecteur sera dimensionné avec des dimensions par défaut de 300 x 150 pixels.
    6. Ouvrez les outils de développement de votre navigateur et explorez le code HTML pour trouver le div balise qui entoure le lecteur. Vous verrez le div tag a une classe de video-js. le div qui contient le joueur aura toujours une classe de video-js . video-js est le nom de classe dont nous aurons besoin lors du stylisme du lecteur
    7. Vérifiez la capture d'écran ci-dessous pour voir la classe en surbrillance en rouge. La balise vidéo, qui est Brightcove Player, est surlignée en vert.

      Remarque : Selon le système d'exploitation et le navigateur utilisés, la version de secours Flash du lecteur peut s'afficher. Si votre lecteur utilise la version de secours Flash du lecteur, vous ne verrez pas video marque. Au lieu de cela, vous verrez le div entourant un objectcomme indiqué ci-dessous.

    8. Maintenant que vous connaissez la classe correcte à utiliser pour un sélecteur CSS, nous pouvons utiliser ce sélecteur de classe pour modifier la taille du lecteur. le style le bloc ci-dessous doit être placé juste au-dessus de l'extrémité du head section de votre page HTML. Dans la terminologie CSS, ceci est un exemple d'utilisation d'un sélecteur de classe. 
              <style>
          .video-js {
            height: 344px;
            width: 610px;
          }
        </style>
      </head>
    9. Enregistrez et parcourez votre page HTML et vous verrez que le lecteur a une nouvelle taille.

    Sélecteur de bouton de lecture

    Une autre approche va être utilisée pour trouver le sélecteur pour le bouton de lecture. Les étapes suivantes vous montrent comment trouver le bon sélecteur pour le bouton de lecture, puis modifier son apparence.

    1. Parcourez votre page HTML.
    2. Faites un clic droit sur le bouton de lecture qui superpose le joueur et sélectionnez Inspecter l'élément.
    3. Notez que le div La balise en surbrillance représente le bouton de lecture et a une classe vjs-big-play-button.
    4. Modifiez vos styles pour inclure du CSS qui change la couleur et l'opacité du bouton de lecture (lignes 17-18) et centre le bouton à l'intérieur du lecteur (lignes 19-23). 
            <style>
          .video-js {
            height: 344px;
            width: 610px;
          }
          .vjs-big-play-button {
            background-color: red;
            opacity: .3;
            top: 0px;
            left: 0px;
            right: 0px;
            bottom: 0px;
            margin: auto;
          }
        </style>
    5. Parcourez votre page et vous verrez que tous les styles n'ont pas été utilisés. Le bouton n'est pas rouge.
    6. Regardez à droite de vos outils de développement et vous verrez un résumé de vos styles. Vous pouvez regarder vers le bas et voir que changer l'arrière-plan en rouge est remplacé, représenté par le fait qu'il est barré.
      Remarque : Dans Safari, le style peut ne pas être barré même s'il est remplacé.

      Ce qui s'est passé, c'est que de nombreux changements sont apportés à la couleur d'arrière-plan. Le style que vous définissez a été remplacé dans la partie « cascade » de CSS. Vous pouvez changer cela en utilisant la fonctionnalité de CSS appelée spécificité. La spécificité détermine quelle règle CSS est appliquée par les navigateurs et est un type de pondération qui a une incidence sur la façon dont vos règles de feuille de style en cascade (CSS) sont affichées. En utilisant la spécificité, vous pouvez rendre votre style plus important que d'autres, en le forçant à être utilisé. Il y a une brève introduction à la spécificité CSS dans le Personnaliser l'apparence du joueur - Legacy Skin doc, et de nombreuses références dans toute recherche.

    7. Notez que le div tag entoure le joueur a de nombreuses classes qui lui sont associées.
    8. Dans ce cas, en utilisant la combinaison de video-js et vjs-mouse "boostera" vos styles pour qu'ils priment. Modifiez vos styles pour utiliser le nouveau sélecteur de combinaison. Notez qu'il n'y a pas d'espace entre les deux premières classes, mais il y en a un avant la dernière classe (des détails sur les raisons peuvent être trouvés dans les règles de spécificité). 
                .video-js.vjs-mouse .vjs-big-play-button {
            background-color: #ff0000;
            opacity: .3;
            top: 0px;
            left: 0px;
            right: 0px;
            bottom: 0px;
            margin: auto;
          }
    9. Parcourez votre page et voyez que tous les styles sont maintenant appliqués.
    10. Encore une fois, regardez à droite de vos outils de développement et vous verrez un résumé de vos styles. Maintenant, vous voyez que votre changement de couleur d'arrière-plan n'est pas remplacé.

    Incorporer dans la page

    Lorsque vous travaillez avec les sélecteurs et les styles ci-dessus, les résultats ont été utilisés avec l'implémentation d'intégration dans la page du lecteur, comme illustré ici:

          <!doctype html>
      <html>
      
      <head>
          <meta charset="UTF-8">
          <title>Embed in page tester</title>
         <style>
          .video-js {
            height: 344px;
            width: 610px;
          }
          .video-js.vjs-mouse .vjs-big-play-button {
            background-color: red;
            opacity: .3;
            top: 0px;
            left: 0px;
            right: 0px;
            bottom: 0px;
            margin: auto;
          }
         </style>
      </head>
      <body>
      
        <video
          data-video-id="4443311217001"
          data-account="1507807800001"
          data-player="30a9c9e0-295b-4102-a7af-a3f86bba91ab"
          data-embed="default"
          class="video-js" controls></video>
        <script src="//players.brightcove.net/1507807800001/30a9c9e0-295b-4102-a7af-a3f86bba91ab_default/index.min.js"></script>
      
      
      </body>
      </html>

    C'est une façon très courante d'utiliser CSS avec une page HTML. Vous voyez le style bloc est sur la page avec le video marque.

    Remarque : En fonction de votre expérience CSS, vous vous demandez peut-être pourquoi le code ci-dessus n'a pas attribué de id attribut au video marque. Vous pouvez alors utiliser un sélecteur comme #myPlayer .vjs-big-play-button. En fait, vous auriez pu et ça aurait marché. Le problème est que le CSS créé à l'aide des sélecteurs d'id ne fonctionnera pas lors de l'utilisation de l'implémentation du lecteur iframe, donc seuls les noms de classe existants ont été utilisés. En n'utilisant pas d' id attribut, les styles seront directement applicables aux implémentations de lecteur iframe.

    iframe

    L'implémentation iframe, sans surprise, place le Brightcove Player dans une iframe que vous placez ensuite dans une page HTML. Vous pouvez styliser l'iframe lui-même, modifier sa taille par exemple, en utilisant des styles sur la page qui contient l'iframe, mais les styles de page de l'iframe NE CASCADE PAS dans l'iframe lui-même. L'iframe extrait littéralement une page HTML séparée, donc les styles souhaités pour le contenu de l'iframe doivent être associés à la page HTML de l'iframe.

    Lors de l'utilisation de l'implémentation iframe de Brightcove Player, les étapes pour appliquer CSS sont les suivantes :

    • Placer les styles dans un fichier accessible à Internet
    • Lier la feuille de style au lecteur (effectué à l'aide de Studio ou de l'API Player Management)
    Remarque : Les étapes suivantes supposent que vous allez travailler à partir du code HTML que vous avez terminé dans les étapes précédentes et indiqué dans l'extrait de code ci-dessus. Si vous souhaitez garder cela comme exemple de travail, copiez simplement le code dans un autre fichier et commencez les étapes suivantes dans le nouveau fichier.
    1. Dans Studio, publiez une vidéo et copiez le code pour une implémentation iframe du lecteur.
    2. Dans votre page HTML, remplacez le code d'intégration dans la page par le code iframe.
    3. Changer la .video-js sélecteur de classe vers un sélecteur d'élément en utilisant iframe comme élément. Partir width et height le même, comme indiqué ici: 
        iframe {
      height: 344px;
      width: 610px;
    }
    4. Vérifiez que votre page HTML ressemble à ce qui suit : 
            <!doctype html>
      <html>
      <head>
        <meta charset="utf-8">
        <title>iframe tester</title>
      
         <style>
          iframe {
            height: 344px;
            width: 610px;
          }
          .video-js.vjs-mouse .vjs-big-play-button {
            background-color: red;
            opacity: .3;
            top: 0px;
            left: 0px;
            right: 0px;
            bottom: 0px;
            margin: auto;
          }
         </style>
      <body>
      
        <iframe src='//players.brightcove.net/1507807800001/30a9c9e0-295b-4102-a7af-a3f86bba91ab_default/index.html?videoId=4443311217001'
          allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>
      
      </body>
      </html>
    5. Parcourez la page et vous verrez que le lecteur iframe est correctement dimensionné. Comme mentionné précédemment, les styles du bouton ne fonctionneront PAS.
    6. Créez un fichier nommé customize-qs.css et enregistrez-le dans un emplacement accessible à Internet.
    7. Coupez les styles du bouton et collez-les dans le fichier et enregistrez-le. Coupez uniquement les styles du bouton, vous ne pouvez pas inclure le style Mots clés. Le fichier ne doit contenir que ce contenu : 
            .video-js.vjs-mouse .vjs-big-play-button {
        background-color: red;
        opacity: .3;
        top: 0px;
        left: 0px;
        right: 0px;
        bottom: 0px;
        margin: auto;
      }
    8. Vérifiez que votre page HTML ressemble à ce qui suit : 
            <!doctype html>
      <html>
      <head>
        <meta charset="utf-8">
        <title>iframe tester</title>
      
        <style>
          iframe {
            height: 344px;
            width: 610px;
          }
        </style>
      <body>
      
        <iframe src='//players.brightcove.net/1507807800001/30a9c9e0-295b-4102-a7af-a3f86bba91ab_default/index.html?videoId=4443311217001'
          allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>
      
      </body>
      </html>
    9. Dans Studio, allez dans le module Players et modifiez le lecteur que vous utilisez pour ce démarrage rapide.
    10. Localisez le PLUGINS et cliquez sur le Éditer bouton.
    11. Clique le CSS et entrez le chemin d'accès à votre fichier CSS, comme indiqué ici:
    12. Cliquez sur sauver pour enregistrer vos modifications dans le PLUGINS section.
      Le lecteur de prévisualisation en haut de la page sera mis à jour pour afficher les styles CSS appliqués au bouton.
    13. En haut de la page, cliquez sur Publier pour publier votre lecteur.
    14. Retournez à votre navigateur et actualisez la page. Une fois les modifications publiées actives, vos styles s'appliqueront au lecteur iframe. Il peut s'écouler plusieurs minutes avant que les modifications publiées ne soient visibles.

    Explorer les styles

    Dans cette section, un JSBin est utilisé pour vous permettre d'explorer les styles et de personnaliser davantage le gros bouton de lecture que ce qui est montré ci-dessus. Une fois que vous obtenez un look que vous aimez, vous pouvez simplement copier le CSS généré et l'utiliser avec l'implémentation de l'un ou l'autre joueur comme indiqué ci-dessus.

    Un problème qui peut sembler déroutant est que la flèche réelle dans le bouton n'est pas une image mais une partie d'une famille de polices. C'est pourquoi vous contrôlez sa taille en utilisant font-size et utilisez ems comme unités.
