Création d'un journal Charles

{{ page.description }}

Charles est un proxy Web qui s'exécute sur votre ordinateur et vous permet d'enregistrer les données envoyées et reçues entre votre navigateur Web et le serveur. À l'aide de Charles facilite le diagnostic et la résolution des problèmes. Pour plus de détails, consultez le Charles proxy Web placer.

Lors de l'utilisation Charles , fermez autant d'applications que possible et fermez toutes les fenêtres de navigateur inutiles pour limiter la quantité d'informations capturées par Charles.

Étapes pour créer un Charles Journal

Les étapes suivantes vous guident tout au long de l'utilisation du Charles proxy Web pour créer un journal enregistré.

  1. Télécharger et installer Charles sur votre appareil. Charles est compatible avec Windows, Mac et Linux. Pour des détails spécifiques à la plate-forme sur l'installation Charles , voir le Charles Installation Remarques.

  2. Sur votre ordinateur, exécutez Charles. Dans le menu, sélectionnez Proxy > Paramètres proxy. Assurez-vous qu'un port a été attribué (par exemple, 8888) et le Activer le proxy HTTP transparent l'option est sélectionnée.
  3. Sélectionnez OK pour enregistrer les modifications et fermer la boîte de dialogue.
  4. Installez le Charles Certificat SSL. Dans le menu, sélectionnez Aider> Proxy SSL> Installer Charles Certificat racine.
  5. dans le Ajouter un certificat boîte de dialogue, sélectionnez Ajouter.
  6. Accès au trousseau s'ouvrira. Dans la liste des certificats, recherchez le Charles Procuration... certificat. Double-cliquez dessus pour voir des informations détaillées.
    Si vous n'utilisez pas macOS, consultez le Charles Documentation pour plus de détails sur les certificats SSL.
  7. Élargir le Confiance section. Pour le Lors de l'utilisation de ce certificat option, sélectionnez Faites toujours confiance.
  8. Fermez la fenêtre du certificat et entrez votre mot de passe administrateur pour mettre à jour les paramètres.
  9. Activer le proxy SSL dans Charles. Dans le menu, sélectionnez Proxy > SSL Proxy Settings. Assurez-vous que Activer le proxy SSL l'option est sélectionnée. Dans le Emplacement rubrique, sélectionnez le Ajouter bouton.
  10. Dans la boîte de dialogue Modifier l'emplacement , accédez au champ Port et entrez 443. Sélectionner D'accord pour enregistrer cette modification.
  11. Sélectionner D'accord pour fermer le Modifier l'emplacement dialogue.
  12. Sélectionner d'accord pour sortir du Paramètres de proxy SSL dialogue.
  13. Il est recommandé de redémarrer votre ordinateur à ce stade pour vous assurer que Charles récupère les nouveaux paramètres de certificat.
  14. Garder Charles ouvert pendant que vous effectuez les étapes suivantes.
  15. Collez l'URL sur la page contenant le lecteur Brightcove que vous dépannez dans la barre d'adresse du navigateur, puis appuyez sur Entrée pour charger la page.
  16. Exécutez les étapes que vous avez précédemment utilisées pour reproduire l'erreur ou le problème. Charles enregistrera automatiquement ce qui se passe.
  17. Si le trafic réseau est nécessaire lorsque le lecteur est en train de lire une vidéo, démarrez la vidéo.
  18. Après avoir reproduit l'erreur, retournez à Charles. Sélectionnez le Structure afficher et trouver l'URL de votre lecteur.
  19. Sélectionner Arrête d'enregistrer.
  20. Enregistrez la session en cours dans un fichier.
  21. Envoyez un e-mail au Charles dossier de session au personnel de soutien approprié.

Ajouter des en-têtes Akamai

Dans cette section, vous apprendrez comment ajouter des en-têtes Akamai à vos requêtes.

  1. De la principale Charles fenêtre, sélectionnez Outils puis Récrire.
  2. dans le Réécrire les paramètres boîte de dialogue, sélectionnez Activer la réécriture.
  3. Ajouter un nouvel ensemble appelé En-têtes Akamai.
  4. Ajoutez une nouvelle règle.

  5. dans le Réécrire la règle boîte de dialogue, sélectionnez un type de Ajouter un en-tête.

  6. dans le Remplacer section, ajoutez ce qui suit:

    Nom: Pragma

    Valeur:

          akamai-x-get-request-id, akamai-x-cache-on, akamai-x-cache-remote-on,
      akamai-x-check-cacheable, akamai-x-get-cache-key, akamai-x-get-extracted-values,
      akamai-x-get-nonces, akamai-x-get-ssl-client-session-id, akamai-x-get-true-cache-key,
      akamai-x-serial-no
  7. Sélectionner D'accord , et alors D'accord.
