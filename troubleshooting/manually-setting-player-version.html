Définition manuelle de la version du lecteur

{{ page.description }}

Pourquoi modifier manuellement la version

Vous devrez peut-être revenir à une version antérieure du joueur car vous trouvez qu'une mise à niveau de lecteur est à l'origine d'un problème. Vous pouvez revenir à une version précédente jusqu'à ce que le test soit terminé pour trouver la raison du problème et qu'il soit résolu.

Pour une discussion approfondie sur les paramètres de version du lecteur dans Studio, consultez le Réglage du mode et de la version de mise à jour du lecteur document.

Processus

Ces étapes vous permettent de définir manuellement une version de lecteur.

  1. Tout d'abord, vous voudrez définir le lecteur pour qu'il soit mis à jour manuellement. Cela empêchera une modification de version d'un joueur d'être remplacée par une mise à jour du joueur. Pour ce faire, allez dans Studio, JOUEURS module et dans le Informations sur le joueur section définir le Mode de mise à jour à Manuel. Bien sûr, enregistrez vos modifications.
  2. Vous allez maintenant définir manuellement la version du lecteur à l'aide du Éditeur JSON dans le JOUEURS module. Ceci est un exemple de configuration d'un lecteur regarde avant tout changement de version est effectué :
    Noter la "locked": true l'attribut est le résultat de l'étape 1.
    dans le Éditeur JSON , modifiez le version attribuer à la version souhaitée. Assurez-vous d'enregistrer vos modifications, puis de publier votre lecteur.

N'oubliez pas une fois le problème corrigé pour changer votre joueur pour être automatiquement mis à jour.
