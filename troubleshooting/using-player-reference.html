Utilisation de la référence du lecteur

{{ page.description }}

Présentation

Cette page est destinée aux clients potentiels et actuels pour effectuer des tests. Voici quelques-unes des options que vous pouvez tester:

  • Mode HTML et mode Flash
  • Peau Luna vs Peau Graphite
  • Différents types de formats vidéo

La page de référence du joueur se trouve sur http://players.brightcove.net/3303963094001/player-reference-app/index.html.

Fournir une source vidéo

La première chose à faire est de fournir une source vidéo pour le lecteur ou d'utiliser l'une des vidéos par défaut fournies. Vous collez l'URL, puis cliquez sur les options de la barre de boutons de configuration avec lesquelles vous souhaitez tester. Vous devez pré-sélectionner les options puis cliquer sur Charge.

La sélection d'options dans la barre de boutons de configuration pendant la lecture n'a aucun effet. Vous devez cliquer sur les boutons de configuration, puis cliquer sur Charge encore. En outre, vous devez choisir manuellement le bon Type de contenu URL pour la vidéo fournie. Si vous choisissez le mauvais type, soit vous obtiendrez une erreur, soit la vidéo ne sera pas lue.

Si vous le souhaitez, vous pouvez utiliser une source Video Cloud pour les tests. Rendez-vous simplement sur le MÉDIAS module dans Studio, cliquez sur une vidéo, puis dans le Fichiers vidéo section cliquez avec le bouton droit sur une entrée, puis choisissez Copier le lien. Vous pouvez ensuite coller le lien copié dans la zone de saisie de l'URL.

Lorsque vous fournissez une source vidéo DASH, il vous sera également demandé de fournir l'URL du serveur de licences. Notez qu'à ce stade, vous ne pouvez pas fournir l'intégralité de keySystemsOptions objet comme Options de lecteur.

Fournir des options de joueur

Une option de la barre de boutons de configuration qui nécessite des explications est la Options du joueur bouton.

Ce bouton vous permet de transmettre certaines options de configuration au lecteur à l'aide de données valides au format JSON. Par exemple, ce qui suit fournirait une nouvelle affiche et une boucle vidéo à la fin:

{ "poster": "http://solutions.brightcove.com/fish_poster.png", "loop": true }

le Options disponibles pour le code intégré avancé (sur la page) document fournit des options que vous pouvez utiliser. Notez que width et height Les options ne fonctionneront pas correctement en tant qu'options du lecteur car elles sont remplacées par la configuration de la page.

Affichage des données affichées

Les données rapportées sur la page sont généralement faciles à interpréter, deux domaines seront expliqués un peu plus ici, ceux étant le Commutation du débit binaire et Chronologie de l'événement sections.

Commutation du débit binaire

Premièrement, si vous regardez une source vidéo MP4, vous ne verrez aucune information dans cette section. Pour un MP4, la source vidéo est envoyée et il n'y a plus de négociation. Pour HLS, des segments sont périodiquement demandés, donc en fonction de la bande passante, des segments de qualité inférieure ou supérieure peuvent être demandés. Une ligne orientée vers le haut (pente positive) dans le graphique signifierait que le débit disponible augmente et des segments de meilleure qualité seront demandés. Inversement, une ligne orientée vers le bas (pente négative) dans le graphique signifierait que le débit disponible diminue et que des segments de moins bonne qualité seront demandés.

Les lignes du graphique correspondent aux rendus disponibles pour la vidéo. Lorsque le graphique tracé se déplace au-dessus d'une ligne, le joueur demandera des segments de meilleure qualité. Le contraire est vrai lorsque le graphique tracé se déplace sous une ligne. Notez que vous ne verrez probablement pas de changement immédiat de la qualité vidéo lorsque le graphique du traceur franchit une ligne. Les segments demandés devraient être envoyés sur la base de la demande et, lorsqu'ils sont reçus, ajoutés aux segments actuellement dans la mémoire tampon.

Remarque : Le graphique ci-dessus montre une bande passante disponible élevée et non variable.

La chronologie des événements n'affichera que les événements qui ont été distribués. Cette capture d'écran montre une vidéo HLS avant qu'elle ne commence à jouer, et seule la progress et loadedmetadata les événements ont été envoyés.
